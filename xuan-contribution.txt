My contributions to this project covered the front end of the pipeline, specifically the database design and the entity resolution work that Phase 1 and Phase 2a depended on. The starting point was a CSV scraped from layoffs.fyi with about 4,300 rows that had inconsistent capitalization, missing fields, percentage values stored as text, and duplicate company entries with subtle spelling drift. None of that was usable downstream, so I built the schema and ingestion logic that gave the rest of the pipeline something reliable to query against. The schema lives in sql/schema.sql and consists of five tables in third normal form: companies, layoff_events, daily_prices, market_index, and event_windows. I chose 3NF over a single wide table to avoid update anomalies, and I used SERIAL surrogate keys instead of tickers as primary keys since tickers can change (FB became META, SQ became XYZ). Data quality is enforced inside the database itself through CHECK constraints, foreign keys, UNIQUE constraints, and indexes on the columns that downstream queries hit hardest. The ingestion script ingest_layoffs.py handles the rest. It normalizes column names to snake_case, strips whitespace from text fields, parses dates with errors=coerce so a single malformed row cannot kill the whole job, casts percentage strings to floats, and deduplicates by keeping the most complete record per company. Every insert uses ON CONFLICT DO NOTHING, which makes the script fully re-runnable. Phase 2a was the harder problem. Mapping company names to stock tickers turns into an entity resolution puzzle very quickly because a naive fuzzy match will silently bind Apple to Apple Hospitality REIT. I solved this with a layered strategy: a hand curated override map of about 130 companies, then yfinance Search with a token_set_ratio threshold of 85, then direct yfinance validation as a fallback. The result was 219 matched companies, covering roughly 56 percent of total layoff impact. I also authored the Phase 1 and Phase 2a documentation in documentation/log.txt.
